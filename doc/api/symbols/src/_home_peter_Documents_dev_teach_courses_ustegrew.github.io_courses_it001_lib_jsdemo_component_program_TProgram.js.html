423 [1]: Variable shadowing
424 
425     Regards function local object reference - In this code, is kObj 
426     unique for each instantiated object, or is it the same across 
427     all objects? 
428 
429     function T (n)
430     {
431         var kObj = this;
432         this.n = n;
433 
434         this.Dump = function ()
435         {
436             console.log (this.n);
437             console.log (kObj.n);
438         };
439     }
440 
441     var x, y, z;
442     x = new T(1);
443     y = new T(2);
444     z = new T(3);
445 
446     x.Dump ();
447     y.Dump ();
448     z.Dump ();
449 
450 
451 
452     If it's the same, we expect the output:
453         1
454         3
455         2
456         3
457         3
458         3
459 
460     If it's truly local, we expect the output:
461         1
462         1
463         2
464         2
465         3
466         3
467 
468     Result is:
469         1
470         1
471         2
472         2
473         3
474         3
475 
476     which is GOOD news! It means we can use this local variable technique 
477     to force a constant reference to 'this' in any method, even if methods
478     are called from other contexts. We just need to 
479         * declare a local variable inside the constructor and set it to 'this'
480         * use that local variable in the object's methods, instead of 'this'
481 
482     This technique is very useful when calling methods from other contexts,
483     e.g. DOM element event handlers, within the dojo framework, ...
