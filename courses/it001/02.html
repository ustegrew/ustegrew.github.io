A tour of the Javascript language

Javascript is a scripting computer language, most commonly used within web browsers. It is a dynamic language which means that it performs many tasks during runtime (dynamically) which static languages perform during compile time. For example, a Javascript program can change itself which is very difficult to do with static languages. The Javascript syntax is influenced by the C language, and it supports many features of other programming languages.
Javascript runs mostly within a web browser, but it also is used in PDF documents and even as part of server side web applications.

Concepts

Javascript supports many of the concepts that underlie modern programming languages:

Statements
A statement expresses some action to be carried out. It will often change the internal state of a program, e.g. when a variable value is changed. There are different kinds of statements, for example assignments, or, function calls.
Here are some example statements: 
var n;                          /* Variable declaration                  */
var r = 20;                     /* Variable declaration and definition   */
n = 100;                        /* Simple assignment                     */
n = 2 * r;                      /* Assignment with multiplication term   */
r = Math.sqrt (16)              /* Assignment with function call         */
if (r == 10) r = 0;             /* Control flow statement                */
window.alert ("Hello World");   /* Show message box                      */
For more info, see the chapters on statements.
Variables
A variable is a container for a value. That value might be a number, a string, or some other object. A variable always has a particular variable type. This is the kind of data held by that variable. For example, a variable of type integer holds a whole number whilst a variable of type string holds a sequence of characters. 
a = 5;          /* a will be of type integer and hold the value 5       */
b = "Hello";    /* b will be of type string  and hold the value "Hello" */
For more info, see the chapters on variables.
Operators and terms
Like in mathematics, an operator indicates a specific operation on one or more operands. A term is a combination of one or more operations. For example, in the term 2 + 3 the addition operator '+' denotes the addition of the two operands, 2 and 3 yielding the result 5.
For more info, see the chapters on operators and expressions.
Expressions
An expression is a combination of values, operators and functions producing a value. For example, the expression y=2*a would double the value stored in the variable a and store the result in the variable y. The concept of expressions is borrowed from algebra. Examples: 
x = 10;                     /* Assign value 10 to variable x             */
t = "SomeString";           /* Assign value "SomeString" to variable t   */
y = 2 * x * x  +  32;       /* Compute polynomial value 2*x^2 + 32,      */
                            /* store result in variable y                */
z = computemyvalue (23);    /* Call user defined function computemyvalue */
                            /* passing 23 as parameter and assign result */
                            /* to variable z                             */
For more info, see the chapters on operators and expressions.
Control flow
The control flow of a program refers to the order in which the individual statements are executed. A program may just contain a list of statements; these will be executed in the order they are written. Most times a program will contain statements which change the execution sequence. For example, the if statement changes the execution sequence, depending on a condition. There are other statements which change the flow of execution, such as loops or breaks. 
a = 10;
b =  0;

/* Conditional execution */
if (a < 5)
    b = 1;
else
    b = 2;

/* b will be 2 */   
window.alert ("b = " + b);
For more info, see the chapters on flow control.
Structured programming
Structured programming organizes a program into blocks of code. This increases clarity and readability of the program's source code. For example, consider this program written in unstructured code in BASIC: 
a=1000
point1:
if a<=1 then goto end
if (a/2)=int(a/2) then goto point2
a=3*a+1
goto point1
point2:
a=a/2
goto point1
end:
print "Finished!"
Rather humorously, this kind of code is also called "spaghetti code". Compare it with this equivalent structured BASIC code: 
a=1000
while a >= 2
    if (a/2) = int(a/2) then
        a=a/2
    else
        a=3*a+1
    end if
wend
print "Finished!"
Functions
A function is a unit of code which performs a specific task. This unit can then be used in programs wherever that particular task needs to be performed. Often a function can also be used to compute some value and make it available to the program using the function. Without the programming tool of functions there would be much repetition of code (and errors), resulting in much extra work. The use of functions can significantly reduce the cost of developing and maintaining computer programs. Example: 
function CheckValue (x)
{
    if (x < 10)
        window.alert ("Love the number " + x + "!");
    else
        window.alert ("You doofus! I hate the number " + x + "!");
}

CheckValue (5);       /* Bring up message box showing 'Love the number 5!'                */
CheckValue (20);      /* Bring up message box showing 'You doofus! I hate the number 20!' */
Without function, the program would have looked like this: 
x = 5;
if (x < 10)
    window.alert ("Love the number " + x + "!");
else
    window.alert ("You doofus! I hate the number " + x + "!");

x = 20;
if (x < 10)
    window.alert ("Love the number " + x + "!");
else
    window.alert ("You doofus! I hate the number " + x + "!");
For more info, see the chapters on functions.
Javascript has got some additional features found in fewer programming languages:
Dynamic types
Like in most scripting languages, Javascript uses dynamic data types. For example, a variable x can be defined as a number and later rebound to a string: 
var a = 3;          /* a will be an integer                                */
a = 3.54;           /* Now it's a floating point variable!                 */
a = "Hello"         /* Whoopsie! Now it's a string 'This is my number: 15' */
For more info, see the chapters on variables.
Classes and objects
Classes and objects touch the area of object oriented programming. An object is a structure loaded into the computer's memory which combines data and behaviour in one unit. It's a "piece of data which can do things." A class is the recipe which tells the computer how to create a particular object in memory. It's a construct which contains variables and functions. A class is to an object what a cooking recipe is to a meal.
Interpreted language
Javascript always runs as an interpreted language which means that Javascript source code is directly executed by an interpreter without being compiled beforehand.

Syntax

Like any language, Javascript has rules what to write where. These rules are called the syntax rules of the language. Even more than in written natural languages it is important that these rules are followed carefully if we want the program to work as intended. The computer doesn't have the intelligence to guess what some code means if the code isn't written properly! If Javascript code doesn't follow the syntax rules then the program will either fail completely or show unexpected behaviour. Here's an example of good and faulty syntax: 

/* Syntax correct */
a = 3;

/* Faulty syntax (missing semicolon at the line end) */
a = 3
For more info on syntax, see the chapter on statements.

Practically...

Javascript is very easy to use! All that is needed is a text editor to write the program code and a web browser to execute it. The setup for this course is a bit more elaborate, but that is not a requirement to produce Javascript programs.
