Operators & Expressions

Like with a mathematical formula an expression is a statement which calculates some value and stores the result in a variable. An expression has an equal sign = with a variable to its left and a term to its right: 

{variable} = {term}

The term is what calculates and returns the value, and equal sign = tells the system to store the result of the calculation in the variable. The term can consist of one or more constants, variables, function calls or operators. Here some examles: 

var a, alpha, b, c, d, e, f;
alpha = 1.2;

a = 3;                              /* Simple assignment */
b = 2 + 5;
c = a * 4 + 28;
d = Math.random ();                 /* Function call     */
e = Math.sin (alpha * 3.14 / 180);
See Javascript debugger: Exec_01 ()
Be aware that the understanding of the term "expression" in this course deviates from the understanding of "expression" in computer science! See the discussion in the appendix.

Expressions are not equations! You cannot place a term at the left hand side of the = sign! A statement like this one would be legal as a mathematical equation, but illegal in Javascript: 

a + b = 23

You also cannot place more than one variable at the left hand side of the = sign. This statement would be illegal: 

x1, x2 = Math.sqrt (25);     

/* Square root of 25 has two solutions in mathematics (-5, +5), */
/* but Javascript calculates only one of them (+5)              */
Expressions borrow ideas from mathematical formulas.

Operators

An operator represents an operation on some values. That is, the operator stands for something you want to do with the values. For example, the term 3 + 4 stands for add 3 and 4, the + stands for the addition operation.

Operators are the verbs of expressions
Javascript does have more operators than the ones shown here. We just present those operators which can be found in other programming languages, such as Java, PHP, or C++. If you want to know more information specific to Javascript, see the MDN.

Comparison operators

                

                    A comparison is a relational operator which test some kind of relation between
                    two operands.
                

                

                    A comparison always returns a boolean value: true when 
                    the relation holds, and false when it doesn't. For example, the 
                    comparison for equality between the operands 4 and 4 would 
                    return true, because 4 is equal to 4.                
                

                

                    Since the return value of a comparison operator is a boolean value it
                    can be assigned to a variable. The variable can be used like any other 
                    variable.

var a = 10;
var b =  5;
var c;
var d;

c = (a >  b);
d = (a == b);
console.log (a + " >  " + b + ": " + c);
console.log (a + " == " + b + ": " + d);

Generic

                            converts both operands to the same type before performing 
                            the test on (un) equality. This makes it possible to test 
                            operands of different types for equality, e.g. a string 
                            and an integer.
                        

                        
Strict

                        

                            (=== and !==). The strict operator

Conditional operator

                

                    The conditional operator returns one of two values depending on the result
                    of a boolean term. It is often used as a shortcut for the if
                    statement. The conditional operator is used as follows:
                


condition  ?  first-term : second-term


                

                    
    
                        
  • 
                            The condition is treated as a boolean expression and evaluated
                            to either true or false.
                        
    • 
                        
  • 
                            If the condition evaluates to true, return result of first-term.
                        
    • 
                        
  • 
                            If the condition evaluates to false, return result of second-term.
                        
    • 
                    

                


var a =  5;
var b = 20;
var y;

y = (a > 10) ? "Wow, big value!" :  "My, what a small value!";
console.log (y);

y = (b > 10) ? "Wow, big value!" :  "My, what a small value!";
console.log (y);

Operator precedence

                

                    Operators aren't always executed in the order in which they appear in the program. For example,
                    the multiplication has priority before addition.

Brackets

                

                    Like in mathematics, brackets change the order of calculations in a term. They override the 
                    normal operator precedence.
                


a = 2 + 3 * 4       /* a will be 14 */
b = (2 + 3) * 4     /* b will be 20 */
