Operators & Expressions

Like with a mathematical formula an expression is a statement which calculates some value and stores the result in a variable. An expression has an equal sign = with a variable to its left and a term to its right: 

                    {variable} = {term}

The term is what calculates and returns the value, and equal sign = tells the system to store the result of the calculation in the variable. The term can consist of one or more constants, variables, function calls or operators. Here some examles: 

                    a = 3;                              /* Simple assignment */
                    b = 2 + 5;
                    c = a * 4 + 28;
                    d = Math.random ();                 /* Function call     */
                    e = Math.sin (a * 3.14 / 180);
Be aware that the understanding of the term "expression" in this course deviates from the understanding of "expression" in computer science!

Expressions are not equations! You cannot place a term at the left hand side of the = sign! A statement like this one would be legal as a mathematical equation, but illegal in Javascript: 

                    a + b = 23

You also cannot place more than one variable at the left hand side of the = sign. This statement would be illegal: 

                    x1, x2 = Math.sqrt (25);     

                    /* Square root of 25 has two solutions in mathematics (-5, +5), */
                    /* but Javascript calculates only one of them (+5)              */
Expressions borrow ideas from mathematical formulas.

Operators

An operator represents an operation on some values. That is, the operator stands for something you want to do with the values. For example, the term 3 + 4 stands for add 3 and 4, the + stands for the addition operation.

Operators are the verbs of expressions

Javascript does have more operators than the ones shown here. We just present those operators which can be found in other programming languages, such as Java, PHP, or C++. If you want to know more information specific to Javascript, see the MDN.

Comparison operators

A comparison is a relational operator which test some kind of relation between two operands.

A comparison always returns a boolean value: true when the relation holds, and false when it doesn't. For example, the comparison for equality between the operands 4 and 4 would return true, because 4 is equal to 4.

Since the return value of a comparison operator is a boolean value it can be assigned to a variable. The variable can be used like any other variable. 

                    a = 10;
                    b =  5;
                    c = (a >  b);       /* true  */
                    d = (a == b);       /* false */

With regards to the (un) equality operators, there are two flavours:

Generic
(== and !=). The generic operator converts both operands to the same type before performing the test on (un) equality. This makes it possible to test operands of different types for equality, e.g. a string and an integer.
Strict
(=== and !==). The strict operator does not do any type conversion. Thus, both, the types and values of both operators are being considered when doing the test.

String operators

There are two operators which chain strings together to a new string. This operation of chaining strings is called concatenation.

Operator Operation Examples
+ concatenate 
                                a   = "opera"
                                b   = "tor";
                                y1  = a + b;         // "operator"
                                y2  = a + " " + b;   // "opera tor"
+= concatenate (compound) 
                                a = "My "
                                b = "operator ";
                                c = "is ";
                                d = "not ";
                                e = "working.";

                                y  = a;            // "My "
                                y += b;            // "My operator "
                                y += c;            // "My operator is "
                                y += d;            // "My operator is not "
                                y += e;            // "My operator is not working."

Strings can also be concatenated with variables of other types. Variables of other types will be converted to strings during the concatenation. 

                    a = "I told you ";
                    b = 123456;
                    c = " times ";
                    y = a + b + c + ": Stop exaggerating!";

Conditional operator

The conditional operator returns one of two values depending on the result of a boolean term. It is often used as a shortcut for the if statement. The conditional operator is used as follows: 

                    condition  ?  first-term : second-term

  • The condition is treated as a boolean expression and evaluated to either true or false.
  • If the condition evaluates to true, return result of first-term.
  • If the condition evaluates to false, return result of second-term. 

                    a = ( 5 > 10)  ?  "Wow, big value!"  :  "My, what a small value!";
                    b = (20 > 10)  ?  "Wow, big value!"  :  "My, what a small value!";
For more on special operators, see the section "Special Operators" in the MDN

Operator precedence

Operators aren't always executed in the order in which they appear in the program. For example, the multiplication has priority before addition. 

                    2 + 3 * 4           // yields 14, not 20

Brackets

Like in mathematics, brackets change the order of calculations in a term. They override the normal operator precedence. 

                    a = 2 + 3 * 4;      /* a will be 14 */
                    b = (2 + 3) * 4;    /* b will be 20 */
