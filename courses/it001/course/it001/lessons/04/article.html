Statements

A statement is an instruction to the computer, it makes the computer "do" something. Statements are the individual instructions in the program, and they are executed in the order in which they appear. This order is called the flow of execution. Programs aren't limited to a linear flow. Javascript provides flow control statements which change the execution order.

A computer program is a list of statements.

Kinds of statements

There are different kinds of statements, mainly for manipulation of data, in/output and and flow control.

Declaration/Definition

A declaration announces the existence of a variable to the system, and a definition assigns a value to the variable. Once a variable is declared and defined it can be used. 

                    /* Variable x has not been declared yet. */
                    console.log ("Variable x before declaration: " + x);

                    /* Variable x is declared. */
                    var x;
                    console.log ("Variable x after declaration: " + x);

                    /* Variable x is defined. */
                    x = 3;
                    console.log ("Variable x after definition: " + x);
For more info, see the chapter on variables.

Expression

Like with a mathematical formula an expression is a statement which calculates some value and stores the result in a variable. An expression has an equal sign = with a variable to its left and a term to its right: 

                    {variable} = {term}

This allows for calculations with a notation similar to the one used in mathematics. 

                    var x = 0;
                    var y = 0;

                    /* Expression for a parabola f(x) = 3*x*x + 2 with x = 23 */
                    x = 23;
                    y = 3 * x * x + 2; /* The expression */

                    console.log ("f(x) =  3*x*x + 2     ---->     f (" + x + ") = " + y);
For more info, see the chapter on operators.

Function call

A function call is an instruction which passes control to a sub program (the function). Control returns to the calling program once the sub program has finished. Along with the function call the caller can pass arguments, i.e. data needed by the sub program. The sub program can return a value back to the caller. 

                    function a ()
                    {
                        console.log ("Executing function a ()");
                    }

                    console.log ("About to call function a ()");
                    a ();
                    console.log ("Returned from function a ()");
For more info, see the chapter on functions.

Control flow

A program executes statements in the order in which they appear in the program, unless the order of execution is changed by flow control statements. Flow control statements are what enables the program to react appropriately to given conditions. 

                    var x = 10;
                    var b =  0;

                    if (x >= 5)
                    {
                        b = 1;
                    }
                    else
                    {
                        b = 2;
                    }

                    console.log (b);
For more info, see the chapter on control flow.

Comment

Comments are used to convey information for the developers to make the source code easier to understand. Comments will be ignored when the program is executed.
Comments come in two flavours - the one line comment and the multiline comment.

The one-line comment

The one line comment starts with a double slash // and is valid from the double slashes to the end of the line. Code to the left of the double slashes will be executed. 

                    // This is a one line comment

                    var a = 3;
                    // var b = 4;   Commenting out a line is a great debugging tool!

                    console.log ("Type of a: " + typeof (a));
                    console.log ("Type of b: " + typeof (b));

The multi-line comment

The multiline comment starts with one forward slash followed by an asterisk /* and ends with an asterisk followed by a slash */. Everything in between will be ignored when the program is executed. Multiline comments can span any number of lines, from a single line to as many as there are in between the start symbol /* and the end symbol */. This makes multiline comments more flexible than single line comments. 

                    /* A multiline comment used on one line */

                    /* A multiline comment
                       spanning
                       several
                       lines
                     */

                    var a = 3; /* This will execute */

                    /*
                    Commenting out a chunk of code is a great debugging tool!

                    var b = 4;
                    var iWantThisVariable = 5;
                     */

                    console.log ("Type of a: " + typeof (a));
                    console.log ("Type of b: " + typeof (b));
                    console.log ("Type of iWantThisVariable: " + typeof (iWantThisVariable));

Blocks

Statements can be grouped inside a block. Blocks are enclosed in braces { } and usually span multiple lines. It's good practice to indent all statements within the braces 

                    { /* Start of statement block */
                        b = 5;
                        c = 6;
                        console.log ("hello");
                    } /* End of statement block */

Statement blocks are commonly used with control flow statements. 

                    if (a == 5)
                    {   /* Start of statement block */
                        b = 6;
                        c = 6;
                        console.log ("hello");
                    }   /* End of statement block */

Syntax

Like any language, Javascript has a few rules when it comes to the way we have to write our programs. Those rules are called the syntax of the language. Even more than in written natural languages it is important that these rules are followed carefully if we want the program to work. The computer doesn't have the intelligence to guess what some code means if the code isn't written properly!

Statements

Statements must end with a semicolon (otherwise you may get a syntax error and the program will not work). The following statements need a semicolon at the end:

Expression 
                                a = 10;
                                y = x + 28;
                                z = Math.sin (y);
                                n = 28 * Math.random ();
Declaration 
                                /* Declaration */
                                var a;

                                /* Definition */
                                var b = 3;
do... while 
                                do
                                {
                                    /* do something */
                                }
                                while (someCondition);
break 
                                if (a == 5)
                                    break;

Clauses

Clauses end without semicolon. A clause always has to be followed by a statement or block. For example, the if statement starts as an if clause containing the condition and is followed by a statement or block (which is the stuff done when the condition is fulfilled).

if 
                                if (someCondition)
                                    /* Followed by a statement or block */
else 
                                if (someCondition)
                                    /* Followed by a statement or block */
                                else
                                    /* Followed by a statement or block */
for 
                                for (i = 1; i <= n; i++)
                                    /* Followed by a statement or block */
while 
                                while (someCondition)
                                    /* Followed by a statement or block */
Blocks 
                                if (someCondition)
                                {               /* No semicolon after opening brace */
                                    a = 3;
                                    b = 5;
                                    c = 7;
                                }               /* No semicolon after closing brace */

A few clauses end on a colon. These are the case and default clause in a switch.

switch
(and friends) 
                            switch (i)
                            {
                                case 0:
                                    /* Do something */
                                default:
                                    /* Do something else */
                            }
For more specifics on the different clauses, see the chapter on control flow.

Case sensitive

Javascript is a case sensitive language. This means that identifiers are different if their capitalization is different. This concerns all identifiers - Javascript keywords, variables, functions and so on. Examples of non working code due to wrong capitalization: 

                    /* Wrong capitalization when using variable 'myVariable' */
                    /* Message box will show "undefined"                     */
                    var myVariable = 10;
                    console.log (Myvariable);


                    /* Wrong capitalization on 'console' */
                    Console.log ("Hello World");


                    /* Wrong capitalization when calling function 'Print' */
                    function Print (message)
                    {
                        console.log (message);
                    }
                    print ("Hello World");


                    /* Mispelled keyword 'var', leads to syntax error */
                    vAr x;

Whitespace

Whitespace is the term for all spaces, tabs and newlines outside string constants. Javascript ignores whitespace in most cases. These statements all do the same: 

                    b = 3;              /* This line does the same as the next line */
                    b    = 3;

Statements can be broken up into several lines (e.g. to increase readability), but this should be done with caution. This example will fail with a syntax error, on the first line, because "this is my is seen as an unterminated string constant: 

                    var a = "This is my 
                    very
                    very
                    long sentence";

If the string has to be broken up by newlines, it's better to write: 

                    var a = "This is my " +
                            "very " +
                            "very " +
                            "long sentence";

Line breaks can have unwanted side effects: 

                    function a()
                    {
                        return
                                15;
                    }

                    var b = a ();
                    console.log (b);

This will show a message box saying "undefined", not the expected "15". The program will work when we remove the newline after return: 

                    function a()
                    {
                        return 15;
                    }

                    var b = a ();
                    console.log (b);
