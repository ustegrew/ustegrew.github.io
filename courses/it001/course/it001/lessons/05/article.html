Variables

In mathematics, a variable is a named placeholder for some quantity. In computing, a variable is a named container for information. In most cases, this information is simply data such as numbers or text. A variable is like a bucket with a name written on it (variable name) and something in it (value).

Variables store the information needed by the program.

Variable names

As a variable is a named container we need to note the rules applying to variable names. These rules apply to names in other contexts - most notably, function names. We won't get into all the details of proper names in Javascript, but present a rule of thumb which represents good practice across several programming languages:
A name should

  • Start with a letter or underscore:
        a .. z, A .. Z, _
  • Followed by a sequence of letters, digits and underscores:
        a..z, A..Z, 0..9, _

Examples:

Good x, freezeLevel, OppositeCorner, hello2, _key
Bad (starts with digit '5') 5Elements
Bad (illegal character '-') node-length
Bad (contains space[s]) delete new node

Declaration and definition

Before a variable can be used it must be declared and defined.

Declaration

The declaration announces the existence of the variable to the Javascript system. As soon as the Javascript system finds a declaration it sets storage aside for the resp. variable. This is like getting the bucket out of the cupboard. You declare a variable with the keyword var. 

                    var x;

You could omit the keyword and simply assign the value, but this can introduce problems and should not be used. 

                    x = 3;
Always use var when declaring variables.

Definition

A variable can't be used if it's just declared. We say that the variable is undefined. Just declaring an variable is like getting an empty bucket out. An empty bucket offers no useful content. To make the variable useable we need to set it to some initial value. This setting to some initial value is called the definition. 

                    /* Declare variable x. */
                    var x;


                    /* 
                     * At this point, the variable x is just declared, 
                     * but not yet defined.
                     */
                    console.log ("x: " + x);


                    /* Define variable x to have the value 3 */
                    x = 3;


                    /* 
                     * Now the variable x is defined.
                     */
                    console.log ("x: " + x);

Variable declaration and definition can also be done in one statement. 

                    var x = 3;

                    /* Variable x is declared AND defined. */
                    console.log ("x: " + x);
Always declare and define variables before you use them.

Literals

A literal is a fixed value in source code. Such fixed data is needed very often. Example uses would be definitions, or, calculations. 

                    x = 3;                              /* number literal in assignment     */

Variable types

Variables are classified by their variable type, depending on which data they hold.

Primitive types

Most programming languages provide basic, inbuilt data types. Such types are called primitive types. Javascript has got five primitive types.

Numbers Integers such as -5, 10, or, floating point numbers such as -4.2, 3.14
String Textual data, such as "Hello World". The textual data is always delimited by double quotes ".
Logical Boolean values, either true or false.
null A special keyword denoting an empty value, A variable can be emptied by assigning it the value null.
undefined A special value reserved for variables that have not been defined.

Composite types

Many languages provide data types in which one or more primitive elements are bound together in one unit. Such types are called composite types. The area of composite types has a lot to do with the area of object oriented programming.

Object oriented programming is beyond the scope of this course, so we won't handle it in any detail.

Weakly typed

Javascript does not fix any variable as to which type it has. The type of a variable depends on its content. This means, when we change a variable's content during run time it may change it's type with it. This makes Javascript a weakly typed language. 

                    var x = 3;
                    console.log ("x = 3,         type of x is: " + typeof (x));

                    x = 3.2;
                    console.log ("x = 3.2,       type of x is: " + typeof (x));

                    x = "Hello";
                    console.log ("x = 'Hello',   type of x is: " + typeof (x));

                    x = true;
                    console.log ("x = true,      type of x is: " + typeof (x));

                    x = undefined;
                    console.log ("x = undefined, type of x is: " + typeof (x));

                    x = null;
                    console.log ("x = null,      type of x is: " + typeof (x));

The "weak types" feature is particular to the Javascript language, as well as some others. There are languages which are strongly typed, such as C++. In C++, trying to assign a string to an integer variable will result in a compiler error: 

                    void TestFunc ()
                    {
                        int x;

                        /* Compiler error: invalid conversion from 'const char*' to 'int' */
                        x = "Hello";
                    }

Arrays

There are tasks for which simple variables aren't sufficient. Here some example tasks:

  • Sorting a list of integers.
  • Finding the average over a number of measurements.
  • Searching for an entry in a list of names.
  • Finding the shortest route between two cities.

All these tasks have in common that they need to perform a particular operation on a group of related data items (e.g. a list of numbers). The resp. task becomes even harder to solve when the number of items is unknown at programming time! One of the structures which makes group operations possible is the array.

An array is...

An array is an ordered, named, indexable collection of elements with a finite amount of data. In detail: An array is...

A collection
An array is a group of unique elements.
Ordered
Elements stay in the same sequence for the lifetime of the array, unless changes are done programmatically.
Named
The whole group has one name which is shared by all its elements.
Indexed
Each element in the array has a unique index.
Finite
There is always a finite number of elements in an array.

All elements in an array have got the same variable name, and each element has a unique index.

This bucket array would be...

A collection
There are multiple buckets in one lot.
Named
The whole lot is called x.
Indexed
Each bucket has a unique integer index.
Ordered
The buckets stay in the order they've been put.
Finite
There's always a countable number of buckets.

Another comparison to an array would be a street: The street is a collection of houses (the elements), it is named (has a street name), indexed (each house has a number), ordered (houses don't normally get up and walk) and finite (there's no street with an infinite number of houses).

Creating an array

Like all variables, arrays need to be declared and defined. There are different ways of defining arrays in Javascript, but we won't handle them all. The notation we will use here is the square bracket notation. Here we create the Javascript equivalent of the above array of buckets: 

                    var buckets = [10, 25, 16, 45];

This creates a new array called buckets and fills it with four elements in the order in which they appear inside the brackets: 10, 25, 16, 45. This order will not change, unless explicitly done by programming.

Array elements don't all need to be of the same type. It's possible to mix variable types in an array. 

                    var x = [15, true, "I am element #2", null];
                    console.log ("x: " + typeof (x[0]) + ", " + typeof (x[1]) + ", " + 
                                         typeof (x[2]) + ", " + typeof (x[3])          );

It's also possible to create empty array elements by using an empty comma (','). The empty element will be set to undefined. The resulting array is called a sparse array. More about sparse arrays below. 

                    var x = [1, , 2];
                    console.log (x [1]);

Array literals

An Array is defined using an array literal. The literal is a list of values enclosed in square brackets []. 

                    var a = [];                 /* Empty array                                          */
                    var b = [20];               /* Array with  one element                              */
                    var c = [1, "Hello"];       /* Array with two elements, each of a different type    */
                    var d = [ [1, 2], [5, 6] ]; /* Array with two sub arrays as elements (table)        */

Accessing array elements

Array elements can be accessed by index. The index is an integer number and has to be written in square brackets []. The first element has the index 0 (zero), the second index has the value 1 and so on. The last element has the index of one less than the number of elements stored in the array. 

                    var x = [10, 25, 16, 45];
                    console.log ("x: " + x [0] + ", " + x [1] + ", " + x [2] + ", " + x [3]);

                    x [0] = 98765432;
                    console.log ("x: " + x [0] + ", " + x [1] + ", " + x [2] + ", " + x [3]);

Sparse arrays

In Javascript, it is possible to create sparse arrays. These are arrays with holes in them. In the following example we are creating an array with a hole at position 1 (second element). This means that the second element will be undefined. 

                    var x = [];     /* Creates an empty array                   */

                    x [0] = 1;      /* Set first array element...               */
                    x [2] = 100;    /* Oops, we're skipping the second element! */
                                    /* Now, our array has a hole                */
                                    /* at position #1 !                         */
Sparse arrays are used in Javascript and some other languages. We just mention then as a matter of information - and won't treat them in more detail.

Getting the size of the array

One often needed piece of information about an array is how many elements it has stored. This is provided by the length property. 

                    var x = [5, 1, 3, 7, 2, 9, 3];
                    console.log ("number of elements stored in array x: " + x.length);
